Can a variable declared by let, in a function still exists after the function finishes running?

NO, let does not care about global or local or whether you’re inside a function or not. Instead, let cares about the block you’re in

Can var variable declared by var, in a function still exists after the function finishes running?

NO, You can only access var variables that have the same scope or a higher scope.
So, from inside a function, you can access the global scope. But, globally, you can’t access a variable that is created only in a function.
